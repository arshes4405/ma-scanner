▶ 매수 조건 (전체 AND)
----------------------------------------------------------------
  1. 현재봉 양봉
  2. 직전봉 음봉
  3. 직전봉 RSI(14) < 티어별 임계값
  4. 현재봉 RSI < (직전봉RSI + 2 + floor(경과분/10))
  5. 현재가 < 직전봉 (고가+저가)/2
  6. 직전봉 (저가+종가)/2 < 볼린저 임계(20,2) — 티어별 fromLower
  7. 24시간 거래량 >= $1,000,000
  8. 언랭 코인은 거래량 상위 250개만 스캔

----------------------------------------------------------------
▶ 매매 설정
----------------------------------------------------------------
  [매수]
  - 알트 신규 : 티어별 금액 | 30x Isolated (실패시 20x → 10x)
  - 알트 DCA  : 다음 봉 재신호 시 초기금액 10% 추가, 티어별 최대
  - 메이저    : $10,000 | 50x Cross (실패시 20x), DCA 없음

  [SL/TP - positionMonitor]
  - 반익절 전 : -3% 전량 청산
  - +5% 도달  : 절반 매도
  - 반익절 후 : +0.5% 이하 시 나머지 청산
  - notional < $2,000 종목만 관리

  [SL 쿨다운]
  - SL 후 8시간 재매수 금지 (메이저 제외)
  - trade_log.csv의 SL/AUTO_SL 기준으로 판단

▶ 자동 티어 관리 (promoteCoins)
----------------------------------------------------------------
  - 매시 5분 자동 실행 (updateTrades 5분 후)
  - trade_log.csv 전체 분석 → tier_config.json 갱신
  - TIER 기준: TIER1(순익절>=5) / TIER2(>=3) / TIER3(>=1)
  - 블랙 기준: 순익절 <=-2 → EXCLUDE_SYMBOLS 자동 추가
  - SYSTEM SL + MANUAL AUTO_SL 2패스 집계 (중복 방지)

----------------------------------------------------------------
▶ trade_log.csv 스키마
----------------------------------------------------------------
  datetime, symbol, action, entry_price, exit_price,
  qty, pnl_pct, pnl_usdt, source, order_id

  - SYSTEM : positionMonitor 자동 (SL/TP_HALF/BE_CLOSE)
             order_id = Binance orderId
  - MANUAL : updateTrades 수집 (AUTO_CLOSE/AUTO_SL)
             order_id = SYMBOL_SEC 포맷 (예: ATUSDT_1746403354)
